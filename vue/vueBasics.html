    # 4. 循环（列表）渲染

    1. 如何遍历对象？--- 也可以使用 v-for

      数组：有两个参数：v-for="(item, index) in array"

      对象：有三个参数：v-for="(val, key, index) in object"

    2. key 的重要性。key不能乱写（如 random 或者 index）

    3. v-for 和 v-if 不能一起使用！

      v-for 比 v-if 的计算优先级高一些，一起使用会先遍历，每遍历一个，就会做一次 if 判断，浪费性能。

      可以将 v-if 放到 v-for 所在节点的 父节点 或者 子节点。

    # 5. 事件

    1. event 参数，自定义参数

      <button @click="onClick1">btn1</button>
<button @click="onClick2(2, $event)">btn2</button>

mehtods: {
	onClick1(event) { //... },
	onClick2(val, event) { //... }
}

    2. 事件修饰符，按键修饰符

      事件修饰符：

      <!-- 阻止单机事件继续传播 -->
<a v-on:click.stop="doThis"></a>
<!-- 修饰符可以串联 -->
<a v-on:click.stop.prevent="doThis"></a>
<!-- 提交事件不再重载页面 -->
<form v-on:submin.prevent="onSubmit"></form>
<!-- 只有修饰符 -->
<form v-on:submit.prevent></form>
<!-- 添加事件监听器时使用事件捕获模式 -->
<!-- 即内部元素触发的事件先在此处理，然后才交由内部元素进行处理 -->
<div v-on:click.capture="doThis"></div>
<!-- 只当在 event.target 是当前元素自身时触发处理函数 -->
<!-- 即事件不是从内部元素触发的 -->
<div v-on:click.self="doThis"></div>

      按键修饰符

      <!-- 即使 Alt 火 Shift 被一同按下时也会触发 -->
<button @click.ctrl="onClick"></button>
<!-- 有且只有 Ctrl 被按下时才触发 -->
<button @click.ctrl.exact="onClick"></button>
<!-- 没有任何系统修饰符被按下的时候才触发 -->
<button @click.exact="onClick"></button>

    3. 【观察】事件被绑定到哪里？

      1. event 是原生的
      2. 事件被挂载到当前元素
